### 1、Process 和 Thread
- 程序：指令与数据的有序集合，其本身没有任何运行的含义，是一个静态的概念。
- 进程：执行程序的一次执行过程，它是一个动态的概念，是系统资源分配的最小单位
- 线程：通常一个进程中含有若干线程（一个进程最少含有一个线程），线程是CPU调度和执行的最小单位
<br/><br/>

### 2、Thread 静态代理
**静态代理**
- 1、目标对象和代理对象需要实现相同的接口；
- 2、代理对象代理目标对象，完成一些目标对象无法完成的事情，目标对象专注自己的事情。
Thread 实现了 Runnable 接口，Thread 类中维护了一个目标对象，run 方法调用目标对象的 run 方法。
```java
/* What will be run. */
private Runnable target;

@Override
public void run() {
    if (target != null) {
        target.run();
    }
}
```

### 3、线程的五大状态
- 1、new：Thread t = new Thread(); 线程对象一旦创建就进入到了新生状态
- 2、就绪状态：当调用 start() 方法，线程立即进入就绪状态，但不意味着立即调度执行，取决于CPU调度
- 3、运行状态：就绪状态的现场，当CPU调度其线程运行时，线程才进入执行状态，真正执行线程体的代码块
- 4、阻塞状态：当调用sleep，wait或同步锁定时，线程进入阻塞状态，就是代码不往下执行，阻塞事件解除后，重新进入就绪状态，等待CPU调度
- 5、死亡状态：线程中断或者结束，一旦进入死亡状态，就不能再次启动。
<br/>

### 4、线程方法
- 1、setPriority(int newPriority)：更改线程的优先级
- 2、static void sleep(long millis)：在指定的毫秒数内让当前正在执行的线程休眠
- 3、void join()：等待该线程终止
- 4、static void yield()：暂停当前正在执行的线程对象，并执行其他线程。yield是让线程从运行状态变成就绪状态，然后重新参与调度，所以它不一定就会在其他线程后面执行（yield方法退出到就绪状态，参与调度被执行的概率与其他线程一样）
- 5、boolean isAlive()：测试线程是否处于活动状态
